Market Resolution Request

A prediction market question enters the protocol for verification

Trigger
▼

Sources

Polymarket API, on-chain trigger events, or manual submission via the FastAPI endpoint (POST /resolve).

Input Format

Question ID, market address, resolution criteria, and an optional deadline. The protocol normalizes all queries into a binary yes/no format.

Validation

Checks question hasn't already been resolved, verifies market exists on-chain, and confirms the resolution window is open. Rejects duplicates.

Multi-Agent Research

N agents independently investigate using diverse strategies

Core
▼

Web Search

Multiple search engines queried in parallel. Each agent uses the battle-tested web_resolver module with staggered requests and caching.

API Queries

Price feeds (CoinMarketCap, CoinGecko), sports APIs, election data. Each agent may specialize in different data sources.

Cross-Reference

Every claim requires 2+ independent sources. Temporal verification ensures evidence is from the correct timeframe, not stale data.

Diversity

Agents use different system prompts, temperature settings, and research strategies to avoid correlated errors. Powered by Gemma4 26B on M4 Max.

3

Calibration-Weighted Consensus

Aggregate votes weighted by historical accuracy (Brier scores)

Innovation
▼

Weight Computation

Each agent's weight = 1 / (brier + 0.01) scaled by prediction count confidence. New agents start at base weight until they hit 20 predictions.

Linear Opinion Pool

Weighted average of all agent probabilities. Normalized weights ensure the final consensus is a proper probability between 0 and 1.

Threshold Logic

Consensus > 0.90 resolves YES. Consensus < 0.10 resolves NO. Between triggers a dispute period with extended research time.

def aggregate_consensus(votes, registry): weights, probs = [], [] for vote in votes: record = registry.get(vote.agent_id) w = compute_weight(record.brier_score, record.total_predictions) weights.append(w) probs.append(vote.probability) total = sum(weights) consensus = sum(w/total * p for w, p in zip(weights, probs)) return consensus # e.g., 0.97
4

Calibration Registry (On-Chain)

Permanent record of every agent's prediction accuracy

On-Chain
▼

Brier Score

BrierScore = (1/N) * sum((forecast - outcome)^2). A proper scoring rule where 0 is perfect and 1 is worst. Updated after every resolution.

Reputation Decay

Scores decay 5% per epoch toward 0.25 (random baseline) if an agent stops participating. Prevents stale reputation from dominating.

Tier System

Oracle (<0.10, 3x) → Expert (0.10-0.15, 2x) → Reliable (0.15-0.20, 1.5x) → Standard (0.20-0.25, 1x) → Novice (>0.25, 0.5x)

Soulbound Identity

Each agent gets an ERC-721 soulbound token. Non-transferable, meaning reputation can't be bought or sold. You earn it or you don't.

5

Settlement Contract

On-chain finality, reward distribution, and slashing

Finality
▼

Optimistic Resolution

Result is submitted immediately when consensus exceeds threshold. A challenge period follows where any agent can flag and trigger a second verification round.

Reward Distribution

reward_i = (weight_i / total) * pool * accuracy_bonus. Agents on the wrong side get nothing. Strong incentive to be accurate, not to follow the crowd.

Slashing

Agents must bond tokens to participate. Consistently wrong agents (Brier > 0.35 over 50 predictions) get slashed. Slashed tokens go to the reward pool.

6

Score Update & Feedback Loop

Every resolution makes the next one more accurate

Loop
▼

Brier Score Update

Registry updates each agent's Brier score with the new outcome. Weight derivation recalculated. Agents who were right gain influence; wrong agents lose it.

Training Data

Every prediction + outcome pair becomes DPO training data. Agents learn from their mistakes through preference pair training. The protocol improves on autopilot.

Compound Accuracy

More resolutions = more data = better calibration = more accurate consensus = more markets trust us = more resolutions. The flywheel spins.
